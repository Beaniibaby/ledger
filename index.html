Sync your data automatically across devices via your own Google Drive. Uses the drive.file scope — Ledger can only see files it created in your Drive, never your other content. Requires a one-time setup of a Google OAuth Client ID.

⚠ About sync conflicts: Last write wins. If you edit on two devices before either syncs, the most recently saved version overwrites the other. A local backup snapshot is kept before every sync — use "Pull from Drive" to discard local changes and load the cloud version, or check the warning before doing so.

Google Drive setup guide

This is a one-time setup — about 10 minutes. Your data stays in your own Google Drive, in a single private file only Ledger can see.

Step 1 — Create a Google Cloud project

  1. Go to console.cloud.google.com and sign in with your Google account
  2. Click the project picker at the top, then "New Project"
  3. Name it anything (e.g. "Ledger Sync") and click Create
  4. Wait ~30 seconds for it to create, then make sure that project is selected at the top

Step 2 — Enable the Drive API

  1. In the search bar at the top, type "Google Drive API" and click the result
  2. Click Enable

Step 3 — Configure OAuth consent screen

  1. In the left sidebar (or hamburger menu), go to APIs & Services → OAuth consent screen
  2. Click Get Started if prompted
  3. App name: Ledger (or anything you want)
  4. User support email: your email
  5. Audience: select External
  6. Developer contact: your email
  7. Click through and finish — you don't need to add scopes here
  8. Under Audience → Test users, click Add users and add your own Google email. (Without this you'll get a "blocked" error when signing in.)

Step 4 — Create OAuth Client ID

  1. Go to APIs & Services → Credentials
  2. Click Create Credentials → OAuth client ID
  3. Application type: Web application
  4. Name: anything (e.g. "Ledger Web Client")
  5. Under Authorized JavaScript origins, click Add URI and add the URL where you'll host the file. Important: origins must use HTTPS (except localhost). Examples:
    • https://sites.google.com (if using Google Sites)
    • https://yourname.github.io (if using GitHub Pages)
    • http://localhost:8000 (for local testing)
  6. Click Create
  7. Copy the Client ID (looks like 123456789-abc...apps.googleusercontent.com) and paste it into the field below

Step 5 — Connect

  1. Paste your Client ID into Ledger
  2. Click Connect Google Drive
  3. You'll see a Google warning that the app isn't verified — that's normal for personal apps. Click Advanced → Go to Ledger (unsafe) — it's "unsafe" only because Google doesn't recognize you as the developer; you wrote/own the project
  4. Approve the Drive scope. Done.
Important: The OAuth client only works on the origins you registered in Step 4. If you open the file from your computer (file://), sync won't work. You need to host it somewhere with a real URL. Free options: Google Sites, GitHub Pages, Netlify Drop.
